This was originally created in codepen.io to demonstrate my knowledge of CSS EMs and REMs, and when to use each.

The :root selector font-size is set with a pixel unit. This is the base font-size of the page. Anything that is given a 1rem will be whatever we have the :root font-size set to, in this case it's 16px. This doesn't only affect font-size, we could set the width of an element to be 40rem.

{% highlight css %} :root { font-size: 16px; } {% endhighlight %}

Button 1 has the font-size of 1rem, which would be the font-size of the root. The padding around the button is set to 1rem for the top and bottom, and 1.5rem for the right and left. This means that the padding will stay the same even if the font-size of the button is changed.

Here's the styles for a button that isn't responsive to changing font-size.

{% highlight css %} .button { font-size: 1rem; text-decoration: none; font-weight: bold; color: #e74c3c; border: 1px #e74c3c solid; border-radius: 3px; padding: 1rem 1.5rem; display: inline-block; transition: all 200ms ease-in-out; } .button:hover { color: white; background-color: #e74c3c; } {% endhighlight %} button 1

Button 2 is the same as button 1, but the font-size is increased to 4rem. You can see how the padding of the button hasn't changed.

{% highlight css %} .button { font-size: 4rem; ... } {% endhighlight %} button 2

Button 3 also has a font-size of 4em but the padding is set to 1em for the top and bottom and 1.5em for the right and left. This padding is based off the font-size of the button itself, rather than the root font-size.

{% highlight css %} .button { font-size: 4rem; padding: 1em 1.5rem; ... } {% endhighlight %} button 3

Button 4 shows that the same can be done for the border. Instead of having a fixed size of 1px, we set it to 0.0625em which will make the border thickness be proportionate to the font-size of the element itself.

{% highlight css %} .button { font-size: 4rem; padding: 1em 1.5rem; border: 0.0625em #e74c3c solid; ... } {% endhighlight %} button 4

Now the only thing left to fix is the border-radius. It's a small change and really not that noticeable in this situation but this is just to demonstrate how setting the border-radius to an em value will make it scale with the font-size of that element.

{% highlight css %} .button { font-size: 4rem; padding: 1em 1.5rem; border: 0.0625em #e74c3c solid; border-radius: 0.0625em; ... } {% endhighlight %} button 5

Check out the demo on codepen.io to play around with the EM and REM properties for a better understanding on how they work

REMs are based off the :root font-size. EMs are based off the current element font-size.
